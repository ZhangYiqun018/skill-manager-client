# 四、用户体验与产品完整性

## 总体评价

**产品成熟度**: v0.9 — 核心机制可用，但体验需要打磨才能在 v1.0 与手动管理工作流竞争。**整体 UX 评分: 6.9/10**。导航清晰 8/10，信息层次 7/10，视觉一致性 8/10，但错误处理 5/10 和新手引导 5/10 是最大短板。

## 1. 信息架构评估

**当前导航**: Guide | Library | Discover | Targets | Settings（5 个顶级页面）

### 1.1 "Targets" 概念不透明（HIGH）

"Targets" 是开发者术语，大多数用户配置一次后不再访问。作为顶级导航项，与 Library/Discover 并列显得不合比例。

**建议**: 重命名为 "安装位置" 并合并到 Settings 中。

### 1.2 Library 和 Discover 是两个独立的浏览体验（MEDIUM）

从用户心智模型看，"我的技能"和"可用技能"应该是统一视图的两个筛选维度，而非两个独立页面。

**建议**: 考虑未来合并为统一的 "Skills" 视图，带 Installed / Available / All 筛选标签。

### 1.3 Guide 页面不应是永久导航项（MEDIUM）

引导/新手教程作为永久导航项暗示应用本身不够直观。应改为首次运行向导，完成后从导航移除（或降级为 Settings 中的"查看引导"链接）。

## 2. 用户旅程分析

### 旅程 A: 首次用户打开应用

1. 看到 Guide 页面或空白 Library
2. GuidePage 仅 24 行变更——可能只是文本引导
3. **问题**: 无技能、无目标，核心体验为空
4. **问题**: 无自动检测 Claude Code/Cursor 安装
5. **问题**: 无"快速开始"推荐

**严重性**: HIGH — 首次体验决定留存率。需要交互式设置向导：自动检测安装 → 注册为目标 → 推荐热门技能 → 引导完成首次安装。

### 旅程 B: 发现并安装技能

1. 导航到 Discover → 等待加载 → 浏览平铺列表 → 搜索
2. 点击技能 → 侧面板详情 → 点击安装
3. InstallModal 打开 → 选择目标、方法、作用域 → 确认

**问题**:
- **无一键安装（HIGH）**: 每次安装都需打开 modal 做多个选择。已知技能应有智能默认的"快速安装"按钮
- **无"安装后引导"（MEDIUM）**: 安装完成后只有 toast，无"技能已就绪！试试这个命令"的后续流程
- **无分类浏览（MEDIUM）**: 平铺列表随目录增长越来越难以浏览

### 旅程 C: 管理已安装技能

1. Library → 筛选"已安装" → 点击详情 → 卸载

**问题**:
- **无更新检测（HIGH）**: 远程技能更新后用户无从得知
- **无启用/禁用（HIGH）**: 只能安装或卸载，无法临时停用
- **无批量操作（MEDIUM）**: 无法多选批量卸载/更新

### 旅程 D: 配置安装目标

1. Targets → 目标列表 → 添加表单

**问题**:
- **无目录选择器（HIGH）**: 需手动输入路径（`tauri-plugin-dialog` 已添加但需验证是否使用）
- **无自动检测（HIGH）**: 应自动发现常见 Claude Code/Cursor 配置目录

### 旅程 E: 更改应用设置

Settings 页面仅 43 行变更，可能功能有限。需验证是否包含语言选择器、主题切换、默认目标配置等。

## 3. 功能完整性

### 用户能做的 ✅
- 扫描本地目录发现技能
- 浏览远程技能注册表
- 查看技能详情（6 个标签：Overview, Variants, Content, Files, Installs, Origins）
- 安装/卸载技能（支持方法/作用域/agent 选项）
- 管理安装目标 CRUD
- 举报问题技能
- 切换主题
- 查看新手引导
- 调整设置
- 键盘快捷键（Cmd+K 搜索, Cmd+R 刷新, Cmd+1-5 切换标签）

### 用户不能做但自然期待的 ❌

| 缺失功能 | 严重性 | 影响 |
|---------|--------|------|
| 更新已安装技能 | HIGH | 用户被困在旧版本 |
| 启用/禁用技能 | HIGH | 无轻量管理方式 |
| 一键智能安装 | HIGH | 基础安装摩擦过大 |
| 技能使用量/评分 | MEDIUM | 无社交证明辅助决策 |
| 跨 Library+Discover 统一搜索 | MEDIUM | 搜索体验碎片化 |
| 导出/导入技能配置 | MEDIUM | 无法跨设备分享配置 |
| 技能变更日志/版本历史 | MEDIUM | 无法评估更新内容 |
| 批量操作 | MEDIUM | 管理多技能时效率低 |
| 键盘快捷键文档 | LOW | 快捷键存在但不可发现 |

## 4. UX 一致性问题

- **卡片设计**: Library 和 Discover 的卡片是否使用统一组件？应统一为共享 `SkillCard`
- **安装入口**: Library（右键菜单）vs Discover（按钮），入口不同但 modal 应一致
- **详情面板行为**: 推挤内容还是覆盖？Library 和 Discover 是否一致？
- **术语**: 验证全局一致性——"Install" vs "Add", "Uninstall" vs "Remove", "Skill" vs "Plugin"
- **破坏性操作**: 卸载是否有确认弹窗？删除目标是否警告已安装的技能？

## 5. 具体 UX 问题

### 5.1 LibraryDetailsPanel 承担过多职责（HIGH）

1000+ 行的面板同时充当安装管理器、详情查看器、历史追踪器和操作中心。建议拆分为独立视图或面板内标签。

### 5.2 Discover 页面 5 种输入方式同时可见（HIGH）

扫描磁盘、导入文件夹、搜索注册表、选择预设、配置源——信息过载。建议重新排列优先级（注册表搜索优先，磁盘扫描次之），折叠不常用选项。

### 5.3 Resolution 对话框过于复杂（HIGH）

发现重复/变体技能时，用户突然面临"合并还是创建变体？"的决策，缺少上下文预览。建议先展示差异对比，再让用户决策。

### 5.4 缺少加载骨架屏（MEDIUM）

页面从空白跳到完整内容，无渐进式展示。建议添加内容形状的 skeleton 组件。

### 5.5 空状态缺少引导插图（LOW）

空 Library（"无技能"）和空搜索结果仅纯文本。应添加插图和行动指引（"浏览市场" / "调整搜索"）。

### 5.6 Toast 是唯一的反馈机制（MEDIUM）

安装/卸载等重要操作仅通过自动消失的 toast 反馈。建议对关键操作使用面板内内联状态反馈。

## 6. 产品级评估

**核心价值主张**: "通过 GUI 管理 Claude Code 技能，替代手动编辑配置文件。"

**能让用户选择此工具而非手动管理的关键**:
1. **一键安装** — 当前需要 3+ 次点击和多个选择
2. **可视化技能浏览+分类** — 发现技能是杀手级功能
3. **更新通知** — "3 个技能有更新"是打开应用的理由
4. **技能预览** — 安装前展示技能功能（示例、截图）
5. **配置同步** — "导出我的配置并在另一台机器上导入"

**产品成熟度**: 早期 Alpha。核心机制可用，但体验需大量打磨才能与手动工作流竞争。
